25|## 1. 项目概述
26|
27|IT 工单系统是一个轻量级、自托管的 IT 服务管理平台，用于跟踪、分配和解决 IT 支持请求。系统提供 Web 管理界面和 RESTful API，支持工单全生命周期管理、SLA 追踪、附件管理、数据分析仪表盘等功能。
28|
29|### 核心功能
30|
31|- **工单管理**: 创建、编辑、状态流转、分配、备注
32|- **SLA 管理**: 按优先级设定 SLA 时限，自动计算到期时间
33|- **全文搜索**: PostgreSQL 全文检索（开发环境支持 ILIKE 降级）
34|- **仪表盘**: 多维度统计分析（状态分布、工作量、SLA 达标率、趋势等）
35|- **用户与权限**: 基于角色的访问控制（管理员 / IT人员 / 最终用户）
36|- **附件管理**: 上传、下载、删除附件
37|- **工作时段配置**: 可自定义工作日和工作时间
38|- **密码管理**: 修改密码、忘记密码/重置密码流程
39|- **通知**: SMTP 邮件 + Webhook 双通道通知

377|### 5.2 数据表详细说明
378|
379|#### 5.2.1 `users` — 用户表
380|
381|| 字段 | 类型 | 约束 | 说明 |
382||------|------|------|------|
383|| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
384|| `username` | VARCHAR(50) | NOT NULL, UNIQUE, INDEX | 登录用户名 |
385|| `email` | VARCHAR(255) | NULLABLE | 邮箱地址 |
386|| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 哈希密码 |
387|| `display_name` | VARCHAR(100) | NOT NULL | 显示名称 |
388|| `role` | VARCHAR(20) | NOT NULL, DEFAULT 'end_user' | 角色: `admin` / `it_staff` / `end_user` |
389|| `is_active` | BOOLEAN | DEFAULT TRUE | 账号是否激活 |
390|| `created_at` | TIMESTAMPTZ | NOT NULL | 创建时间 |
391|| `updated_at` | TIMESTAMPTZ | NULLABLE | 更新时间 |
392|
393|**角色说明**:
394|- `admin`: 管理员 — 完全权限，可查看所有工单
395|- `it_staff`: IT 人员 — 处理分配给自己工单
396|- `end_user`: 最终用户 — 仅创建工单
