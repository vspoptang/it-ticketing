## 1. 项目概述

IT 工单系统是一个轻量级、自托管的 IT 服务管理平台，用于跟踪、分配和解决 IT 支持请求。系统提供 Web 管理界面和 RESTful API，支持工单全生命周期管理、SLA 追踪、附件管理、数据分析仪表盘等功能。

### 核心功能

- **工单管理**: 创建、编辑、状态流转、分配、备注
- **SLA 管理**: 按优先级设定 SLA 时限，自动计算到期时间
- **全文搜索**: PostgreSQL 全文检索（开发环境支持 ILIKE 降级）
- **仪表盘**: 多维度统计分析（状态分布、工作量、SLA 达标率、趋势等）
- **用户与权限**: 基于角色的访问控制（管理员 / IT人员 / 最终用户）
- **附件管理**: 上传、下载、删除附件
- **工作时段配置**: 可自定义工作日和工作时间
- **密码管理**: 修改密码、忘记密码/重置密码流程
- **通知**: SMTP 邮件 + Webhook 双通道通知

### 5.2 数据表详细说明

#### 5.2.1 `users` — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
| `username` | VARCHAR(50) | NOT NULL, UNIQUE, INDEX | 登录用户名 |
| `email` | VARCHAR(255) | NULLABLE | 邮箱地址 |
| `password_hash` | VARCHAR(255) | NOT NULL | bcrypt 哈希密码 |
| `display_name` | VARCHAR(100) | NOT NULL | 显示名称 |
| `role` | VARCHAR(20) | NOT NULL, DEFAULT 'end_user' | 角色: `admin` / `it_staff` / `end_user` |
| `is_active` | BOOLEAN | DEFAULT TRUE | 账号是否激活 |
| `created_at` | TIMESTAMPTZ | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMPTZ | NULLABLE | 更新时间 |

**角色说明**:
- `admin`: 管理员 — 完全权限，可查看所有工单
- `it_staff`: IT 人员 — 处理分配给自己工单
- `end_user`: 最终用户 — 仅创建工单

#### 5.2.2 `tickets` — 工单表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
| `ticket_number` | VARCHAR(15) | UNIQUE, INDEX | 工单编号 (如 `TK20260629-001`) |
| `title` | VARCHAR(200) | NOT NULL, INDEX | 工单标题 |
| `description` | TEXT | NULLABLE | 详细描述 |
| `status` | VARCHAR(20) | NOT NULL, DEFAULT 'pending', INDEX | 状态 |
| `priority` | VARCHAR(20) | DEFAULT 'medium' | 优先级 |
| `assignee` | VARCHAR(100) | NULLABLE | 指派人 display_name |
| `category` | VARCHAR(50) | NULLABLE | 分类名称 |
| `creator_name` | VARCHAR(100) | NOT NULL | 创建者名称 |
| `resolution_notes` | TEXT | NULLABLE | 解决方案/备注 |
| `sla_due_at` | TIMESTAMPTZ | NULLABLE | SLA 到期时间 |
| `first_response_at` | TIMESTAMPTZ | NULLABLE | 首次响应时间 |
| `satisfaction` | VARCHAR(20) | NULLABLE | 满意度: `satisfied`/`neutral`/`unsatisfied` |
| `search_vector` | TSVECTOR(PostgreSQL) / TEXT(SQLite) | GENERATED | 全文搜索向量 |
| `created_at` | TIMESTAMPTZ | NOT NULL | 创建时间 |
| `resolved_at` | TIMESTAMPTZ | NULLABLE, INDEX | 解决时间 |
| `updated_at` | TIMESTAMPTZ | NULLABLE | 更新时间 |

**工单状态**:

| 状态值 | 中文 | 说明 |
|--------|------|------|
| `pending` | 待处理 | 新建工单，等待分配 |
| `in_progress` | 处理中 | 已分配，正在处理 |
| `escalated` | 已升级 | 升级至上级处理 |
| `completed` | 已完成 | 处理完成 |
| `cancelled` | 已取消 | 已取消 |

**状态流转规则** (默认):

```
pending ──(any_staff)──▶ in_progress
pending ──(admin_only)─▶ cancelled
in_progress ─(owner)───▶ completed
in_progress ─(admin_only)─▶ cancelled
in_progress ─(owner)───▶ escalated
escalated ──(owner)────▶ in_progress
escalated ──(owner)────▶ completed
escalated ──(admin_only)─▶ cancelled
completed ──(any_staff)─▶ in_progress
cancelled ──(admin_only)▶ pending
```

> 权限说明: `any_staff`=任何IT人员, `owner`=当前指派人, `admin_only`=仅管理员

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | INTEGER | PK, AUTOINCREMENT | 主键 |
| `ticket_id` | INTEGER | FK → tickets.id, ON DELETE CASCADE, INDEX | 所属工单 |
| `event_type` | VARCHAR(50) | NOT NULL | 事件类型 |

## 8. 认证与安全

### 8.1 认证流程

```
1. 用户提交 username + password (POST /auth/login)
2. 服务端验证:
   - 查询用户 (username)
   - bcrypt.verify(password, password_hash)
   - 检查 is_active
3. 生成 JWT:
   - Header: {"alg": "HS256", "typ": "JWT"}
   - Payload: {"sub": user_id, "exp": now + 24h}
   - 签名: HMAC-SHA256(JWT_SECRET_KEY)
4. 设置 HttpOnly Cookie:
   - Set-Cookie: access_token=<jwt>; HttpOnly; SameSite=Lax; Max-Age=86400; Path=/
5. 后续请求自动携带 Cookie
6. 依赖注入提取用户:
   - get_current_user: 解析 JWT → 查询用户 → 验证激活状态
   - require_user: 未登录返回 401
   - require_admin: 非管理员返回 403
   - require_staff: 非 IT 人员返回 403
```

### 8.2 安全措施

| 层级 | 措施 | 说明 |
|------|------|------|
| 传输 | HTTPS + Cookie Secure | 生产环境启用 `COOKIE_SECURE=true` |
| Cookie | HttpOnly | JavaScript 无法访问令牌 |
| Cookie | SameSite=Lax | 防止跨站请求携带 Cookie |
| CSRF | Origin/Referer 校验 | 非 GET/HEAD/OPTIONS 请求验证来源 |
| CSRF | 豁免路径 | API 端点 (`/api/`) 跳过 CSRF 检查 |
| 密码 | bcrypt | 单向哈希，自动加盐 |
| JWT | HS256 | 最小密钥长度 32 字符，带过期时间 |
| 登录 | 频率限制 | 每 IP 每分钟最多 10 次尝试 |
| 密码重置 | 防枚举 | 统一返回成功消息 |
| 权限 | 基于角色 | admin / it_staff / end_user 三级 |
| 工单隔离 | assignee 检查 | 非管理员只能访问分配给自己的工单 |
| 状态转换 | 细粒度权限 | any_staff / owner / admin_only |
| 数据库 | 参数化查询 | SQLAlchemy ORM 防注入 |
| 容器 | 非 root | Docker 以 `appuser` 运行 |
| 日志 | 结构化 | 无敏感信息泄露 |

### 8.3 环境变量安全


- 生产: Gunicorn 多 Worker
  - 推荐: `WORKERS = 2 * CPU核心数 + 1`
  - 默认: 2 (通过 `$WORKERS` 环境变量控制)

### 10.4 资源预估

| 场景 | CPU | 内存 | 磁盘 |
|------|-----|------|------|
| 小型团队 (<10人) | 1 核 | 512 MB | 10 GB |
| 中型团队 (<50人) | 2 核 | 1 GB | 50 GB |
| 大型团队 (100+人) | 4 核 | 2 GB | 200 GB |

### 10.5 开发与生产差异

| 特性 | 开发 (SQLite) | 生产 (PostgreSQL) |
|------|---------------|-------------------|
| 数据库 | SQLite (单文件) | PostgreSQL 16 |
| 全文搜索 | ILIKE 降级 | tsvector + GIN |
| 并发 | 单连接 | 连接池 10+20 |
| 服务器 | uvicorn --reload | gunicorn + uvicorn worker |
| Cookie安全 | COOKIE_SECURE=false | COOKIE_SECURE=true |

## 附录

### A. 工单编号规则

格式: `TK` + `YYYYMMDD` + `-` + 三位序号

示例: `TK20260629-001`

### B. 时区处理

所有时间使用 **CST (UTC+8)** 存储和显示：

```python
# app/timezone_helper.py
CST = timezone(timedelta(hours=8))

def now() -> datetime:
    return datetime.now(CST)
```

### C. SLA 计算逻辑

1. 创建工单时，根据优先级从 `priority_config` 获取 SLA 小时数
2. 仅计算工作日的工作时间（依据 `workday_config`）
3. `sla_due_at = created_at + SLA工作小时`
4. 仪表盘展示 SLA 达标率、逾期工单等

### D. 通知机制

- **SMTP 邮件**: 配置 SMTP 后自动发送（`aiosmtplib` 异步发送）
- **Webhook**: POST JSON 到配置的 URL（`httpx` 异步请求）
- **事件记录**: 所有通知记录在 `notification_events` 表
