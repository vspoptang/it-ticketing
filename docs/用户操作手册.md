## 1. 系统概述

IT 工单系统是一套轻量级、高性能的 IT 服务管理平台，旨在帮助团队高效地**创建、分配、跟踪和解决** IT 支持请求。

### ✨ 核心特性

| 功能模块 | 说明 |
|---------|------|
| 📝 **工单管理** | 创建/编辑/查看/指派/状态流转/附件上传/备注 |
| 📊 **数据仪表盘** | 摘牌排行、热力图、工作负载、SLA 趋势、满意度 |
| ⏱️ **SLA 自动计算** | 按优先级自动计算到期时间，支持工作日/业务小时配置 |
| 🔐 **权限控制** | 基于角色的细粒度状态流转权限 |
| 📧 **通知系统** | SMTP 邮件通知 + Webhook 回调 |
| 🏷️ **分类管理** | 硬件/软件/网络/账号 + 复杂度权重 |
| 🔌 **REST API** | 标准 RESTful 接口，支持第三方集成 |
| 🐳 **Docker 部署** | 一键启动，支持 SQLite / PostgreSQL 双数据库 |

### 🗺️ 系统路由

| 路由 | 功能 | 权限 |
|------|------|------|

## 4. 工单管理

### 4.1 工单状态生命周期

工单从创建到关闭经历以下状态流转：

```
       ┌──────────┐
       │  待处理   │ (pending)
       └────┬─────┘
            │
      ┌─────┴─────┐
      ▼           ▼
┌──────────┐  ┌──────────┐
│  处理中   │  │  已取消   │
│(in_progress)│  │(cancelled)│
└────┬─────┘  └────┬─────┘
     │             │
     │   ┌─────────┘
     ▼   ▼
┌──────────┐  ┌──────────┐
│  已完成   │  │  已升级   │
│(completed)│  │(escalated)│
└────┬─────┘  └────┬─────┘
     │             │
     └── 重开 ─────┤
                   │
         ┌─────────┘
         ▼
    ┌──────────┐
    │  处理中   │
    └──────────┘
```

#### 状态说明

| 状态 | 英文 | 标识色 | 含义 |
|------|------|:--:|------|
| 待处理 | `pending` | 🟡 黄 | 工单已创建，等待处理人接单 |
| 处理中 | `in_progress` | 🔵 蓝 | 有 IT 人员正在处理 |
| 已完成 | `completed` | 🟢 绿 | 问题已解决 |
| 已取消 | `cancelled` | ⚫ 灰 | 工单被取消（可恢复） |
| 已升级 | `escalated` | 🟠 橙 | 需要更高级别处理 |

#### 合法流转

| 当前状态 | 可流转到 |
|---------|---------|
| 待处理 | 处理中、已取消 |
| 处理中 | 已完成、已取消、已升级 |
| 已升级 | 处理中、已完成、已取消 |
| 已完成 | 处理中（重开） |
| 已取消 | 待处理（恢复） |

### 4.2 创建工单

**路径**：`/tickets/new`

1. 在工单列表页点击「➕ 新建工单」
2. 填写表单：

| 字段 | 说明 |
|------|------|
| **标题** | 问题的简要描述，如"打印机无法连接" |
| **描述** | 详细说明，包括环境、复现步骤等 |
| **优先级** | 紧急（4h）、高（8h）、中（24h）、低（48h） |
| **分类** | 硬件 / 软件 / 网络 / 账号 / 其他 |
| **创建人** | 默认使用当前登录用户名 |

3. 点击「提交」创建工单

> 💡 工单创建后自动生成唯一编号，格式为 `IT-YYYY-NNNN`（如 `IT-2025-0001`），并自动计算 SLA 到期时间。

### 5.3 摘牌排行榜 🏆

基于综合评分算法的月度 IT 人员绩效排行：

#### 评分公式

| 维度 | 权重 | 计算方式 |
|------|:--:|---------|
| 📦 加权完成量 | **35** | 按分类复杂度加权后的完成数占比 |
| ⏱️ SLA 达标率 | **25** | 按时完成率 × 25 |
| 🚀 解决速度 | **20** | 相对于最慢者的速度得分 |
| 📞 响应速度 | **5** | 首次响应时间的相对得分 |
| ↩️ 退回扣分 | **-5** | 每件退回扣 1 分，最多扣 5 分 |
| 📥 接收加分 | **+10** | 每接收一件升级工单加 2 分，最多 10 分 |

**总分 = 加权完成 + SLA 达标 + 速度得分 + 响应得分 - 退回分 + 接收加分**

## 6. SLA 管理

### 6.1 SLA 规则

系统根据工单优先级自动计算 SLA（服务等级协议）到期时间：

| 优先级 | SLA 时长 | 说明 |
|:------:|:-------:|------|
| 🔴 紧急 | **4 小时** | 需要立即处理 |
| 🟠 高 | **8 小时** | 高优先级处理 |
| 🟡 中 | **24 小时** | 标准处理时效 |
| 🟢 低 | **48 小时** | 可延后处理 |

> 📌 SLA 时长可在管理后台 `优先级配置` 页面自定义调整。

### 6.2 业务时间计算

SLA 按**业务小时**计算，而非自然小时：

- 📅 **工作日**：周一至周五（可配置）
- 🕗 **工作时间**：08:00 - 17:00（可配置）
- ❌ **非工作时间不计入**：周末、节假日、每日 17:00 后到次日 08:00 前

**示例**：
- 周五 16:00 创建的「紧急」工单（4h SLA）
- 16:00-17:00 = 1h → 剩余 3h
- 下周一 08:00-11:00 = 3h → 截止时间为周一 11:00

### 6.3 SLA 状态标识

| 标识 | 含义 |
|:--:|------|
| 🟢 正常 | 在 SLA 期限内 |
| 🟡 临近 | SLA 即将到期 |
| 🔴 超期 | 已超过 SLA 期限 |

### 6.4 SLA 达标率

仪表盘中的 SLA 达标率 = 按时完成工单数 / 总完成工单数 × 100%

| 流转 | 权限要求 |
|------|---------|

| 待处理 → 已取消 | 管理员专用 |

#### 权限类型

| 类型 | 说明 |
|------|------|
| `admin_only` | 仅管理员可执行 |
| `any_staff` | 任意 IT 人员（admin 或 it_staff）可执行 |
| `owner` | 仅工单的当前指派人（或管理员）可执行 |

> 💡 权限配置支持自定义：管理员可在后台修改任意流转的权限要求。

## 10. 常见操作流程

### 10.1 🔄 标准工单处理流程

```
创建工单 → 指派处理人 → 开始处理 → 添加备注/附件 → 完成工单
```

**详细步骤**：

| 步骤 | 操作者 | 操作 | 说明 |
|:--:|:-----:|------|------|
| 1 | 用户 | 创建工单 | 填写标题、描述、优先级、分类 |
| 2 | IT 人员 | 接单/指派 | 将工单分配给自己或同事 |
| 3 | IT 人员 | 开始处理 | 状态从「待处理」→「处理中」 |
| 4 | IT 人员 | 添加备注 | 记录处理过程和方案 |
| 5 | IT 人员 | 完成工单 | 状态从「处理中」→「已完成」 |

### 10.2 ⬆️ 升级处理流程

```
创建工单 → 开始处理 → 发现问题超出能力范围 → 升级 → 重新指派 → 继续处理 → 完成
```

**详细步骤**：

| 步骤 | 操作者 | 操作 |
|:--:|:-----:|------|
| 1 | IT 人员 A | 开始处理 |
| 2 | IT 人员 A | 升级工单（状态 → 已升级），同时重新指派给 IT 人员 B |
| 3 | IT 人员 B | 开始处理（状态 → 处理中） |
| 4 | IT 人员 B | 完成工单 |

### 10.3 ↩️ 退回/重开流程

**场景一：已完成工单需要重新处理**

```
已完成 → 重开（状态 → 处理中）→ 继续处理 → 完成
```

**场景二：已取消工单需要恢复**

```
已取消 → 恢复（状态 → 待处理）→ 指派 → 开始处理 → 完成
```

### 10.4 👑 管理员配置流程

```
登录 → 管理后台 → 选择配置项 → 修改保存
```

**常用配置任务**：

1. **添加新分类**：`/admin/categories` → 新建 → 填写名称/权重
2. **调整 SLA**：`/admin/priorities` → 修改小时数 → 保存
3. **添加 IT 人员**：`/admin/users` → 找到用户 → 修改角色为 `it_staff`
4. **配置工作日**：`/admin/workdays` → 勾选工作日 / 调整时间

## 11. 常见问题 FAQ

### Q1：如何查看我被指派的工单？

在工单列表页面，使用「指派人」下拉筛选器选择自己的名字即可。管理员默认可以看到所有工单。

### Q2：SLA 时间到了会怎样？

系统会在工单列表中将超期工单标记为红色 🔴，仪表盘中也会统计超期数量。此外，如果配置了邮件通知，系统会发送 SLA 预警邮件。

### Q3：如何修改优先级？

在工单详情页点击「编辑」，修改优先级后保存。注意：修改优先级会**重新计算 SLA 到期时间**。

### Q4：我可以取消别人的工单吗？

普通 IT 人员不能取消工单（即使是指派给自己的）。只有**管理员**可以取消工单。管理员可以在 `权限配置` 页面修改此规则。

### Q5：工单号是怎么生成的？

工单号格式为 `IT-YYYY-NNNN`，其中 `YYYY` 为当前年份，`NNNN` 为当年流水号（从 0001 开始递增）。例如 `IT-2025-0042` 表示 2025 年第 42 个工单。

### Q6：仪表盘的排行榜分数是怎么算的？

排行榜采用综合加权评分，包含 6 个维度：加权完成量（35分）、SLA 达标率（25分）、解决速度（20分）、响应速度（5分）、退回收件（-5分/件，最高-5分）、接收升级（+2分/件，最高+10分）。详见 [5.3 摘牌排行榜](#53-摘牌排行榜-)。

### Q7：什么是"业务小时"？

业务小时仅计算工作日的工作时间内的时长。默认周一至周五 08:00-17:00，共 9 小时/天。周末和下班后的时间不计入 SLA。

例如：周五 16:00 提交的工单，到下周一 11:00 才达到 4 业务小时的 SLA。

### Q8：如何上传附件？

在工单详情页的附件区域点击上传按钮，选择文件即可。支持常见格式（图片、PDF、文本等）。

### Q9：如何评价工单处理满意度？

工单完成后，可以在详情页对处理结果进行满意度评价：满意 😊 / 一般 😐 / 不满意 😞。

### Q10：忘记了密码怎么办？

点击登录页的「忘记密码」链接，输入注册邮箱，系统会发送密码重置链接。如果 SMTP 未配置，请联系管理员手动重置密码。

### Q11：为什么我的仪表盘显示「访问被拒绝」？

仪表盘仅对**管理员**和 **IT 人员**（`it_staff` 角色）开放。如果您是普通用户（`end_user`），请联系管理员升级您的角色。

### Q12：可以批量导入工单吗？

目前系统支持通过 REST API 批量创建工单。请参考 [9. REST API](#9-rest-api) 章节，使用 `POST /api/v1/tickets` 接口。

### Q13：系统支持 PostgreSQL 吗？

是的，系统支持 SQLite（开发/轻量部署）和 PostgreSQL（生产环境）双数据库。通过 `DATABASE_URL` 环境变量切换。

### Q14：如何部署系统？

推荐使用 Docker Compose 一键部署：

```bash
docker-compose up -d
```

系统会自动启动应用服务和数据库（如使用 PostgreSQL），访问 `http://localhost:8000` 即可。

### Q15：如何进行系统健康检查？

访问 `/health` 端点，返回 `{"status": "ok"}` 表示系统正常运行。
