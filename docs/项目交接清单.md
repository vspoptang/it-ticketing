## 四、服务器配置要求

### 4.1 基础设施

| 项目 | 要求 |
|------|------|
| 操作系统 | Ubuntu 20.04+ / CentOS 7+ / Debian 11+ / 任何支持 Docker 的 Linux 发行版 |
| Docker 版本 | 20.10+ |

### 4.2 硬件资源推荐

| 团队规模 | CPU | 内存 | 磁盘 |
|----------|-----|------|------|

| 中型（< 50 人） | 2 核 | 1 GB | 50 GB |
| 大型（100+ 人） | 4 核 | 2 GB | 200 GB |

> **注意**: 以上为服务自身资源需求，不包含操作系统开销。建议预留 20% 余量。

## 五、环境变量配置清单

> **来源**: 由项目根目录 `.env.example` 提取。部署时请复制为 `.env` 并修改。

### 5.1 数据库配置 (必填 ✅)

| 变量名 | 示例值 | 说明 | 必填 |
|--------|--------|------|:--:|
| `DATABASE_URL` | `postgresql+asyncpg://ticketing:your-password@postgres:5432/it_ticketing` | PostgreSQL 连接字符串（含用户名、密码、主机、端口、数据库名）。如使用 Docker Compose，主机名固定为 `postgres`；如使用外部数据库，请填写实际地址。 | ✅ |
| `DB_PASSWORD` | `your-strong-password` | 数据库密码，Docker Compose 启动时用于创建 PostgreSQL 用户 | ✅ |

### 5.2 认证与安全 (必填 ✅)

| 变量名 | 示例值 | 说明 | 必填 |
|--------|--------|------|:--:|
| `JWT_SECRET_KEY` | `至少32个字符的随机字符串` | JWT 签名密钥，**生产环境必须设置，长度 ≥ 32 字符**。生成方式：`python3 -c "import secrets; print(secrets.token_urlsafe(48))"` | ✅ |
| `JWT_ALGORITHM` | `HS256` | JWT 签名算法，默认 HS256，一般无需修改 | |
| `JWT_EXPIRATION_HOURS` | `24` | JWT 令牌过期时间（小时），默认 24 | |
| `COOKIE_SECURE` | `true` | 是否启用 Secure Cookie（仅 HTTPS 传输）。**生产环境必须设为 `true`**（前提是已配置 HTTPS） | |
| `ADMIN_DEFAULT_PASSWORD` | `your-admin-password` | 系统首次启动时自动创建 admin 用户的初始密码。**不设置将随机生成并打印到日志** | ✅ |
| `RATE_LIMIT_LOGIN_PER_MINUTE` | `10` | 登录接口每 IP 每分钟最大请求次数，默认 10 | |

### 5.3 邮件通知 SMTP 配置（可选 📧）

## 七、数据库备份与恢复说明

### 7.1 备份策略

#### Docker Compose 部署（推荐）

```bash
# 1. 备份 PostgreSQL 数据库（完整 dump）
docker exec -it <postgres_container_name> pg_dump -U ticketing -d it_ticketing > backup_$(date +%Y%m%d_%H%M%S).sql

# 2. 备份上传的附件（如有）
docker cp <app_container_name>:/app/app/static/uploads ./uploads_backup_$(date +%Y%m%d)

# 3. 备份 .env 配置文件
cp .env .env.backup_$(date +%Y%m%d)
```

#### 外部 PostgreSQL（不使用 Docker）

```bash
pg_dump -h <host> -U ticketing -d it_ticketing > backup_$(date +%Y%m%d_%H%M%S).sql
```

### 7.2 恢复步骤

```bash
# 1. 停止应用（避免数据写入冲突）
docker-compose stop app

# 2. 恢复数据库
docker exec -i <postgres_container_name> psql -U ticketing -d it_ticketing < backup_YYYYMMDD.sql

# 3. 恢复上传附件（如有）
docker cp ./uploads_backup/. <app_container_name>:/app/app/static/uploads/

# 4. 重启应用
docker-compose start app

# 5. 验证
curl http://localhost:8000/health
```

### 7.3 定期备份建议

| 备份频率 | 备份内容 | 保留策略 |
|----------|----------|----------|
| 每天 | 完整数据库 dump | 保留最近 30 天 |
| 每周 | 数据库 + 附件 + .env | 保留最近 12 周 |
| 每月 | 完整归档 | 永久保留 |

> 💡 建议使用 `cron` 定时任务 + 脚本自动执行备份，并将备份文件同步至异地存储。
