148|
149|| 变量名 | 示例值 | 说明 | 必填 |

247|## 七、数据库备份与恢复说明
248|
249|### 7.1 备份策略
250|
251|#### Docker Compose 部署（推荐）
252|
253|```bash
254|# 1. 备份 PostgreSQL 数据库（完整 dump）
255|docker exec -it <postgres_container_name> pg_dump -U ticketing -d it_ticketing > backup_$(date +%Y%m%d_%H%M%S).sql
256|
257|# 2. 备份上传的附件（如有）
258|docker cp <app_container_name>:/app/app/static/uploads ./uploads_backup_$(date +%Y%m%d)
259|
260|# 3. 备份 .env 配置文件
261|cp .env .env.backup_$(date +%Y%m%d)
262|```
263|
264|#### 外部 PostgreSQL（不使用 Docker）
265|
266|```bash
267|pg_dump -h <host> -U ticketing -d it_ticketing > backup_$(date +%Y%m%d_%H%M%S).sql
268|```
269|
270|### 7.2 恢复步骤
271|
272|```bash
273|# 1. 停止应用（避免数据写入冲突）
274|docker-compose stop app
275|
276|# 2. 恢复数据库
277|docker exec -i <postgres_container_name> psql -U ticketing -d it_ticketing < backup_YYYYMMDD.sql
278|
279|# 3. 恢复上传附件（如有）
280|docker cp ./uploads_backup/. <app_container_name>:/app/app/static/uploads/
281|
282|# 4. 重启应用
283|docker-compose start app
284|
285|# 5. 验证
286|curl http://localhost:8000/health
287|```
