137|## 3. 数据库运维
138|
139|### 3.1 数据库备份
140|
141|#### 3.1.1 使用 pg_dump 备份（通过 Docker）
142|
143|```bash
144|# 备份整个数据库（自定义格式，推荐）
145|docker compose exec -T postgres pg_dump -U ticketing -d it_ticketing -Fc > backup_$(date +%Y%m%d_%H%M%S).dump
146|
147|# 备份整个数据库（SQL 文本格式，可直接阅读）
148|docker compose exec -T postgres pg_dump -U ticketing -d it_ticketing > backup_$(date +%Y%m%d_%H%M%S).sql
149|
150|# 仅备份数据（不含表结构）
151|docker compose exec -T postgres pg_dump -U ticketing -d it_ticketing -Fc --data-only > backup_data_$(date +%Y%m%d_%H%M%S).dump
152|
153|# 仅备份表结构
154|docker compose exec -T postgres pg_dump -U ticketing -d it_ticketing -Fc --schema-only > backup_schema_$(date +%Y%m%d_%H%M%S).dump
155|```
156|
157|#### 3.1.2 备份脚本（推荐）
158|
159|创建 `/opt/scripts/backup_it_ticketing.sh`：
160|
161|```bash
162|#!/bin/bash
163|# IT工单系统数据库备份脚本
164|
165|BACKUP_DIR="/opt/backups/it-ticketing"
166|RETENTION_DAYS=30
167|PROJECT_DIR="/path/to/it-ticketing"
168|
169|mkdir -p "$BACKUP_DIR"
170|
171|cd "$PROJECT_DIR"
172|FILENAME="it_ticketing_$(date +%Y%m%d_%H%M%S).dump"
173|
174|docker compose exec -T postgres pg_dump -U ticketing -d it_ticketing -Fc > "$BACKUP_DIR/$FILENAME"
175|
176|# 压缩备份
177|gzip "$BACKUP_DIR/$FILENAME"
178|
179|# 清理旧备份（保留最近 30 天）
180|find "$BACKUP_DIR" -name "it_ticketing_*.dump.gz" -mtime +$RETENTION_DAYS -delete
181|
182|echo "Backup completed: $FILENAME.gz"
183|```

304|## 4. 故障排查
305|
306|### 4.1 服务无法启动
307|
308|#### 症状：`docker compose up -d` 后容器反复重启或处于 unhealthy 状态
309|
310|**排查步骤：**
311|
312|```bash
313|# 1. 查看容器状态
314|docker compose ps -a
315|
316|# 2. 查看 app 服务日志
317|docker compose logs app --tail=50
318|
319|# 3. 查看 postgres 服务日志
320|docker compose logs postgres --tail=50
321|
322|# 4. 检查端口占用
323|lsof -i :8000
324|lsof -i :5432
325|```
326|
327|**常见原因及解决方案：**
328|
329|| 问题 | 原因 | 解决方案 |
330||------|------|----------|
331|| `Error: Database is not ready` | PostgreSQL 未完成初始化 | 等待几秒后重试，检查 `docker compose logs postgres` |
332|| `Connection refused` | 数据库密码错误 | 检查 `.env` 文件中的 `DB_PASSWORD` |
333|| `Port 8000 already in use` | 端口被占用 | 修改 `.env` 中 `APP_PORT` 或释放端口 |
334|| `JWT_SECRET_KEY must be at least 32 characters` | JWT 密钥太短 | 在 `.env` 中设置至少 32 位的 `JWT_SECRET_KEY` |
335|| `Disk full` | 磁盘空间不足 | 清理磁盘空间（见 4.4 节） |
336|| 镜像构建失败 | 网络问题或依赖冲突 | 使用国内镜像源或检查 `requirements.txt` |
337|
338|### 4.2 数据库连接失败
339|
340|#### 症状：应用日志显示 `could not connect to server` 或 `Connection refused`
341|
342|**排查步骤：**
343|
344|```bash
345|# 1. 确认 postgres 容器正在运行
346|docker compose ps postgres
347|
348|# 2. 检查 postgres 健康状态
349|docker inspect it-ticketing-postgres-1 | grep -A 5 Health
350|
351|# 3. 测试数据库连接
352|docker compose exec postgres pg_isready -U ticketing -d it_ticketing
353|
354|# 4. 测试应用到数据库的网络连通性
355|docker compose exec app python -c "
356|import socket
357|s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
358|s.settimeout(5)
359|result = s.connect_ex(('postgres', 5432))
360|print('Connected' if result == 0 else f'Failed: {result}')
361|s.close()
362|"
363|
364|# 5. 检查数据库进程
365|docker compose exec postgres ps aux | grep postgres
366|```
367|
368|**解决方案：**
369|
370|```bash
371|# 重启数据库服务
372|docker compose restart postgres
373|
374|# 如果问题持续，重建数据库容器（⚠️ 保留数据卷）
375|docker compose down
376|docker compose up -d postgres
377|```
378|
379|### 4.3 页面访问慢
380|
381|**排查步骤：**
382|
383|```bash
384|# 1. 检查系统资源使用
385|docker stats
386|
387|# 2. 检查数据库慢查询
388|docker compose exec postgres psql -U ticketing -d it_ticketing -c "
389|  SELECT pid, now() - pg_stat_activity.query_start AS duration, query, state
390|  FROM pg_stat_activity
391|  WHERE (now() - pg_stat_activity.query_start) > interval '5 seconds'
392|    AND state != 'idle'
393|  ORDER BY duration DESC;
394|"
395|
396|# 3. 检查活跃连接数
397|docker compose exec postgres psql -U ticketing -d it_ticketing -c "
398|  SELECT count(*) AS active_connections FROM pg_stat_activity WHERE state = 'active';
399|"
400|
401|# 4. 检查 Worker 进程数
402|docker compose exec app ps aux | grep gunicorn | grep -v grep
403|
404|# 5. 测试健康检查响应时间
405|time curl -s http://localhost:8000/health
406|```
407|
408|**优化建议：**
409|
410|| 问题 | 解决方案 |
411||------|----------|
412|| CPU 使用率高 | 增加 `WORKERS` 环境变量（建议 = CPU 核数 × 2） |
413|| 内存不足 | 增加服务器内存，或减少 Worker 数 |
414|| 数据库查询慢 | 检查并添加索引，执行 `VACUUM ANALYZE` |
415|| 连接数过多 | 检查是否有连接泄漏，考虑使用连接池 |
416|| 静态资源加载慢 | 配置 Nginx 反向代理 + 静态资源缓存 |
417|
