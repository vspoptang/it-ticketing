# IT工单系统 — 部署包

## 方式一：通用 Docker 部署

```bash
# 1. 克隆项目
git clone https://github.com/vspoptang/it-ticketing.git
cd it-ticketing

# 2. 创建 .env（修改密码）
cp .env.example .env
# 编辑 .env，修改 DB_PASSWORD 和 JWT_SECRET_KEY

# 3. 启动
docker compose up -d

# 4. 验证
curl http://localhost:8000/health
# → {"status":"ok"}
```

## 方式二：群晖 DSM (x86) Docker 部署

> 适用于 x86 架构群晖 NAS（DSM 7.x），通过 Container Manager 或 SSH 部署。

### 2.1 前置准备

| 项目 | 要求 |
|------|------|
| NAS 架构 | x86_64（Intel/AMD），**ARM 机型不支持本指南** |
| DSM 版本 | 7.0 及以上 |
| 套件 | Container Manager（原 Docker）已安装 |
| 存储 | 至少 10GB 可用空间 |
| 网络 | NAS 有稳定内网 IP，开放端口 8000（可自定义） |

### 2.2 生成 x86 部署包（在开发机上操作）

由于开发机可能是 Apple Silicon (ARM)，需强制构建 AMD64 镜像：

```bash
cd /Users/mac/projects/it-ticketing

# ── 方法 A：buildx 多平台构建（推荐） ──

# 1. 创建 buildx 构建器（仅首次）
docker buildx create --name syno-builder --use 2>/dev/null || docker buildx use syno-builder

# 2. 构建 AMD64 镜像并导出为 tar
docker buildx build --platform linux/amd64 -t it-ticketing:latest --load .

# 3. 导出镜像文件
docker save it-ticketing:latest | gzip > it-ticketing-amd64.tar.gz

# 4. 同时导出 PostgreSQL 镜像（群晖上可能需要）
docker pull --platform linux/amd64 postgres:16-alpine
docker save postgres:16-alpine | gzip > postgres-16-alpine-amd64.tar.gz
```

> 产出的两个文件：
> - `it-ticketing-amd64.tar.gz` (约 200MB)
> - `postgres-16-alpine-amd64.tar.gz` (约 100MB)

### 2.3 上传到群晖

```bash
# SSH 方式上传（替换 NAS_IP 为你的群晖 IP）
scp it-ticketing-amd64.tar.gz admin@NAS_IP:/volume1/docker/
scp postgres-16-alpine-amd64.tar.gz admin@NAS_IP:/volume1/docker/
scp .env.example admin@NAS_IP:/volume1/docker/it-ticketing/
```

或在 DSM 网页端通过 File Station 上传到 `/volume1/docker/it-ticketing/`。

### 2.4 群晖上部署（SSH 方式）

SSH 登录群晖后执行：

```bash
# 1. 创建项目目录
mkdir -p /volume1/docker/it-ticketing/data
mkdir -p /volume1/docker/it-ticketing/uploads

# 2. 导入镜像
docker load < /volume1/docker/it-ticketing-amd64.tar.gz
docker load < /volume1/docker/postgres-16-alpine-amd64.tar.gz

# 3. 创建 .env 配置文件
cd /volume1/docker/it-ticketing
cp .env.example .env
vi .env   # 修改以下三个必填项：
          #   DB_PASSWORD=你的强密码
          #   JWT_SECRET_KEY=至少32字符的随机字符串
          #   ADMIN_DEFAULT_PASSWORD=管理员初始密码

# 4. 创建 docker-compose.yml（适配群晖路径）
cat > docker-compose.yml << 'EOF'
services:
  postgres:
    image: postgres:16-alpine
    restart: unless-stopped
    environment:
      POSTGRES_DB: it_ticketing
      POSTGRES_USER: ticketing
      POSTGRES_PASSWORD: ${DB_PASSWORD:-change-me}
    volumes:
      - /volume1/docker/it-ticketing/pgdata:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ticketing -d it_ticketing"]
      interval: 5s
      timeout: 3s
      retries: 5

  app:
    image: it-ticketing:latest
    restart: unless-stopped
    ports:
      - "${APP_PORT:-8000}:8000"
    environment:
      DATABASE_URL: postgresql+asyncpg://ticketing:${DB_PASSWORD:-change-me}@postgres:5432/it_ticketing
      JWT_SECRET_KEY: ${JWT_SECRET_KEY}
      ADMIN_DEFAULT_PASSWORD: ${ADMIN_DEFAULT_PASSWORD}
      UPLOAD_DIR: app/static/uploads
      WORKERS: 2
    depends_on:
      postgres:
        condition: service_healthy
    volumes:
      - /volume1/docker/it-ticketing/data:/app/data
      - /volume1/docker/it-ticketing/uploads:/app/app/static/uploads
EOF

# 5. 启动
docker compose up -d

# 6. 验证
curl http://localhost:8000/health
```

### 2.5 群晖上部署（Container Manager UI 方式）

如果你更习惯用网页界面操作：

**步骤 1：导入镜像**
1. 打开 DSM → Container Manager → 映像
2. 点击「新增」→「从文件新增」
3. 选择 `it-ticketing-amd64.tar.gz` → 确定
4. 同样方式导入 `postgres-16-alpine-amd64.tar.gz`

**步骤 2：创建项目**
1. 切换到「项目」标签页
2. 点击「新增」→ 命名 `it-ticketing`
3. 选择「设置路径」为 `/volume1/docker/it-ticketing/`
4. 选择「创建 docker-compose.yml」→ 粘贴上方 2.4 节中的 compose 内容
5. 点击「下一步」→ 检查环境变量 → 「完成」

**步骤 3：配置环境变量**
1. 在项目详情中点击「操作」→「停止」
2. 编辑 `.env` 文件（File Station → `/volume1/docker/it-ticketing/.env`）
3. 返回 Container Manager → 启动项目

### 2.6 群晖防火墙与外部访问

```bash
# SSH 登录群晖，开放 8000 端口
sudo firewall-cmd --add-port=8000/tcp --permanent 2>/dev/null || \
sudo iptables -I INPUT -p tcp --dport 8000 -j ACCEPT

# 或通过 DSM 网页：控制面板 → 安全性 → 防火墙 → 添加规则
# 端口：8000，协议：TCP，来源：内网 IP 段
```

访问地址：`http://NAS_IP:8000`

> ⚠️ 建议仅在**内网**使用。如需外网访问，请搭配反向代理（群晖自带）并启用 HTTPS。

### 2.7 群晖定期备份

在 DSM 控制面板 → 任务计划中添加定时任务：

```bash
#!/bin/bash
# 每天凌晨 2 点备份数据库
BACKUP_DIR=/volume1/docker/it-ticketing/backups
mkdir -p $BACKUP_DIR
docker exec it-ticketing-postgres-1 pg_dump -U ticketing it_ticketing | \
    gzip > "$BACKUP_DIR/backup-$(date +%Y%m%d).sql.gz"
# 保留最近 30 天
find $BACKUP_DIR -name "*.sql.gz" -mtime +30 -delete
```

| 文件 | 说明 |
|------|------|

```
                    ┌──────────────┐

                    │ Gunicorn+    │
                    │ Uvicorn      │
                    └──────┬───────┘
                           │
                    ┌──────▼───────┐
                    │ PostgreSQL   │
                    │ 16-alpine    │
                    └──────────────┘
```

## 环境变量（必填项）

| 变量 | 说明 |
|------|------|
| `DB_PASSWORD` | PostgreSQL 密码 |
| `JWT_SECRET_KEY` | JWT 签名密钥（≥32字符） |
| `ADMIN_DEFAULT_PASSWORD` | 管理员初始密码 |

## 系统要求

- Docker 24+ & Docker Compose v2
- CPU: 2核+
- 内存: 2GB+
- 磁盘: 10GB+

## 默认账号

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | 由 .env 中 ADMIN_DEFAULT_PASSWORD 设定 | 管理员 |
